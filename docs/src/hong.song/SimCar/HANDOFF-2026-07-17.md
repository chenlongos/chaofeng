## 三、环境与文件系统陷阱

### 1. UNC 写入会产生幻象

Claude 运行在 Windows，代码在 WSL2。禁止用 Write/Edit 直接向 `\\wsl.localhost\...` 新建代码文件。可能出现：

- 工具报告写入成功但磁盘没有文件；
- UNC Read 显示新内容，但 WSL 最终消费者仍读到旧内容；
- commit/reflog 一度显示成功，之后 HEAD 没有前移。

代码写入应走 WSL 单通道，并用最终消费者验证。文档写 D 盘时从 WSL `/mnt/d/...` 单通道写入。

### 2. Python 环境

worktree 没有独立 `.venv`。已安装 funasr/torch 的解释器在：

```text
/root/chaofeng/agent-llm-vla/.venv/bin/python
```

运行 worktree 代码时设置：

```bash
PYTHONPATH=/root/chaofeng/.claude/worktrees/tests-scaffold/agent-llm-vla/src
```

### 3. 代理陷阱

WSL 环境中的代理变量可能指向没有运行的 `127.0.0.1:7890`，导致本地 HTTP 或外网连接失败。启动服务前清除：

```bash
unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY all_proxy ALL_PROXY
```

### 4. pyc 陷阱

测试前清除 worktree 下的 `__pycache__`，再以 pytest 收集和运行结果为准。不要把 UNC Read 或旧 pyc 当成落盘验证。
