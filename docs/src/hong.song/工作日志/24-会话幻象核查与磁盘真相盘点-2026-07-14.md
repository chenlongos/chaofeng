### 2.2 `configs/` 下真实只有一个文件

└── app.yaml    （1491 字节，provider 全是 mock）

### 2.3 config.py 本身没有 bug，也不需要修

真实内容（关键几行）：

```python
DEFAULT_CONFIG_PATH = Path(__file__).resolve().parents[2] / "configs" / "app.yaml"  # 绝对路径

@lru_cache(maxsize=1)
def load_config() -> dict[str, Any]:
    os.environ.setdefault("LEROBOT_HOME", str(Path.home() / "lerobot"))
    path = Path(os.getenv("AGENT_VLA_CONFIG", DEFAULT_CONFIG_PATH))   # line 29
    with path.open("r", encoding="utf-8") as f:                       # line 30 ← 报错点
        return _expand_config_value(yaml.safe_load(f))
```

默认路径是**绝对路径**，正常启动完全没问题。只有当 `run_all_services.sh` 用 `AGENT_VLA_CONFIG` 把它指向一个**不存在的相对路径**（那个 demo 配置）时才会在 line 30 炸。

> 教训：之前几轮我一直在给 config.py "打 fallback 补丁"，那是在**追一个幻影**——文件既没那个 bug，补丁也从没真正落盘（`git status` 显示 config.py 根本没被修改）。

## 六、这次踩到的坑（供以后自保）

1. **上下文压缩会把"计划中/幻象中"的产物写成"已完成"**。恢复会话后，凡涉及"文件是否存在/是否落盘/是否提交"的断言，**一律先用可信通道实证，再动手**。绝不基于摘要断言直接改代码。
2. **本会话的写文件+读回验证通道不可信**（UNC 静默失败 + 读回幻觉）。判断磁盘真伪的硬标准：`wsl.exe` 读出的中文若为原始字节乱码=真读；若整整齐齐=可能是幻象缓存。git 断言一律 `git cat-file -e <sha>` 硬验。
3. **PowerShell/wsl.exe 会吃掉 `$VAR` 和 `for` 循环变量** → 一律字面绝对路径，一次一条命令，不用循环、不用变量。
4. 追 bug 前先确认"这个 bug 的载体文件真的存在且真的是这段代码"，否则可能在修一个不存在的问题（本次 config.py fallback 就是典型）。
